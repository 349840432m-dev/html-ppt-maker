# 动效词汇与质感审计

## 何时读取

生成或修改 HTML 动态 PPT 时默认读取。用户要求"动态效果""高级感""NotebookLM 风""公开课""发布会感"，或描述不清某种动效时，必须用本文件把动效命名、约束和审计标准写进方案。

## 动效先问为什么

动效只服务五件事：

- 建立阅读顺序。
- 说明页面元素之间的空间关系。
- 表达状态变化或反馈。
- 解释复杂结构的生成过程。
- 避免大面积内容突然切换造成割裂。

如果动效只是"显得会动"，删掉。高频操作（键盘翻页、按钮点击、工具栏 hover）必须即时；公开课内容页的揭示可以有 300-700ms，但交互控件控制在 100-200ms。

## 标准词汇

写 deck-plan、动画规划或评审时，用准确术语，避免"高级动一下"这类模糊描述。

- Fade：透明度变化，用于文本、图片、卡片出现或消失。
- Slide：元素从一个方向进入或离开，用于列表、模块和方向性转场。
- Scale：尺寸变化，用于数字、图标或焦点强调；不要从 `scale(0)` 开始。
- Pop in：快速 scale+opacity 进入，适合小组件，不适合正文大段内容。
- Reveal：从被遮挡到可见，适合结论、对照和图表信息逐步打开。
- Stagger：多个相似元素错峰出现，推荐间隔 30-80ms。
- Orchestration：多层元素按主次排好时间线，封面和关键图表页必须有。
- Transform origin：缩放或旋转的起点。
- Origin-aware animation：弹层或标注从触发点附近展开。
- Crossfade：两个状态透明度交叉切换。
- Continuity transition：保持上下文连续的状态切换。
- Morph：形状或结构连续变形。
- Shared element transition：同一元素在页面之间延续位置或身份。
- Layout animation：布局变化被平滑过渡；只在必要时用。
- Direction-aware transition：前进/后退使用相反方向，帮助观众理解导航。
- View transition：页面级连续过渡；PPT 中通常降级为页面转场。
- Press/Tap feedback：点击反馈，按钮用轻微 `scale(.97)` 足够。
- Easing：运动速度曲线。
- Ease-out：快速开始、慢慢停下，UI 入场默认使用。
- Ease-in-out：两端慢、中间快，适合大面积或状态切换。
- Cubic-bezier：自定义速度曲线。
- Asymmetric easing：进入与退出使用不同节奏，避免机械感。
- Spring：带物理感的弹性运动，只用于拖拽、惯性或活体感。
- Bounce：回弹幅度；若使用，保持轻微。
- Momentum / Velocity：拖拽和滑动后的惯性，普通 PPT 不需要。
- Interruptible animation：动画可被下一次操作打断，键盘翻页必须不卡住。
- Loop / Pulse / Float：持续动效；公开课 PPT 中慎用，不能抢正文。
- Blur：轻微模糊可掩盖状态切换，正文最终状态必须清晰。
- Clip-path / Mask：用于揭示、对照、扫描感和局部露出。
- Before-after slider：前后对比页可用，但 PPT 降级为静态对照。
- Line drawing：路径、流程、箭头逐步绘制。
- Number ticker：数字滚动到最终值，必须有来源或明确是假设。
- Tabular numbers：表格数字等宽，防止滚动时跳动。
- Typewriter：逐字出现；公开课中少用，长文本禁用。
- Jank：卡顿或掉帧。
- Compositing / Hardware acceleration：优先用 transform 和 opacity 让浏览器合成层处理。
- Layout thrashing：频繁读写布局造成卡顿，生成 deck 时避免。
- Reduced motion：尊重 `prefers-reduced-motion`，关闭大部分非必要动效。

## 动效参数

CSS token 建议：

```css
:root {
  --ease-out: cubic-bezier(0.23, 1, 0.32, 1);
  --ease-in-out: cubic-bezier(0.77, 0, 0.175, 1);
  --ease-drawer: cubic-bezier(0.32, 0.72, 0, 1);
}
```

执行规则：

- 内容揭示：300-700ms，按叙事重要性决定，不要全页同速。
- 控件反馈：100-200ms。
- Stagger：30-80ms；长列表最多分组错峰，不要拖成排队入场。
- 缩放入场：从 `scale(.94-.98)` 到 `scale(1)`，配合 opacity。
- 禁止 `ease-in` 用于 UI 入场或按钮反馈。
- 禁止 `transition: all`。
- 禁止 `scale(0)`。
- 优先动画 `transform` 和 `opacity`；图表条形可用 `scaleX`，少用 width。
- hover 动效放进 `@media (hover: hover) and (pointer: fine)`。
- 必须提供 `prefers-reduced-motion`。

## 页面类型映射

- 封面：Reveal + Orchestration + Stagger。主标题、注释线、核心视觉分层入场。
- 章节页：Slide 或 Fade，配合 Direction-aware transition；不要堆满小卡。
- 对照页：Reveal、Clip-path、Before-after slider。PPT 降级为左右静态对照。
- 数据页：Number ticker + Tabular numbers + 图表 Stagger。数据必须有来源。
- 流程页：Line drawing + Stagger，路径先走，节点后出现。
- 案例拆解页：Continuity transition 或 Shared element transition 的隐喻，PPT 保留最终拆解态。
- 清单/工作坊页：行级 Stagger，保持节奏快，不能影响阅读。
- 结尾页：简短 Fade/Scale 即可，给行动号召留静止时间。

## PPT 转换边界

HTML 动效是预览和讲授层；`.pptx` 默认只保留静态最终态和页面间转场。需要逐步揭示时，优先拆成多页或用基础 PPT 动画；不能把 HTML 复杂动效承诺为 1:1 转换。

动画规划必须写：

- 使用的标准动效词汇。
- 动效目的。
- HTML 实现方式。
- PPT 转场或降级方式。

## 审计规则

生成 HTML 后做一次质感审计：

| 问题 | 修正 |
|---|---|
| `transition: all` | 改为明确属性，如 `transform, opacity` |
| `ease-in` | 改为 `--ease-out` 或 `--ease-in-out` |
| `scale(0)` | 改为 `scale(.94-.98)` + opacity |
| 动画 width/height/top/left/margin/padding | 改为 transform/opacity，图表条形用 scaleX |
| 没有 `prefers-reduced-motion` | 增加降级规则 |
| hover 未限制设备 | 增加 hover/pointer media query |
| 所有元素同时动 | 增加 30-80ms stagger |
| 动效无法解释目的 | 删除或降级为静态 |

如果用户要求审查动效，输出 `Before | After | Why` 表格，并给出通过/需修改结论。
