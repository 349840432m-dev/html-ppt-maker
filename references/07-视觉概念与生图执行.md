# 视觉概念与生图执行

## 目标

解决“干净但不高级”的问题。用户要求高端、大气、有审美、发布会感、商业大片感或正式公开课时，不能只用卡片、线条、渐变和抽象弧线。必须先做艺术指导，再决定是否生图、生成哪些图、如何进入 HTML/PPT。

## 强制流程

1. 提出 3 个视觉概念方向，每个方向包含隐喻、场景、材质、色彩、动效和适用风险。
1.5 给每个关键概念页指定视觉隐喻：先把抽象概念翻译成可画的图像——增长是"上扬曲线"、瓶颈是"收窄的通道"、战略取舍是"分岔路径"、护城河是"同心圆"——再选版式。禁止跳过隐喻直接落成要点卡片；隐喻想不出来的页才允许用纯文字版式。
2. 选择 1 个方向；若用户没有选择，按主题和受众自行选择最稳妥方向并说明假设。
3. 规划视觉资产清单：封面主视觉、章节视觉、关键概念图、案例图、背景纹理或装饰系统。
4. 如果有 image generation 工具并且用户允许生图，生成或请求生成关键资产；不要停在提示词。
5. 如果不能生图，必须用可编辑图形、摄影占位、图表或抽象系统替代，并在验收中标明降级。
6. 生成 HTML/PPT 前，运行视觉资产审计。

## 三类高级视觉方向

### 1. 商业大片型

- 适合：公开课、路演、发布会、品牌课程。
- 画面：真实或半写实空间、深景深、强光影、少量发光数据节点。
- 材质：玻璃、金属、纸张、屏幕、柔和体积光。
- 风险：容易变成泛科技背景，必须有与主题相关的主体。

### 2. 高级信息图型

- 适合：方法论课程、企业培训、咨询报告。
- 画面：可编辑框架、强网格、精细标注、局部放大、过程路径。
- 材质：纸感、墨线、精密工程图、低饱和强调色。
- 风险：容易太干，需要 3 页以上视觉锚点。

### 3. 叙事场景型

- 适合：用户体验、服务设计、产品流程、案例课。
- 画面：用户动作、任务路径、场景切换、前后对照。
- 材质：插画或半写实，不出现真实可识别人脸。
- 风险：不能伪造真实客户、截图或品牌场景。

## 视觉资产最低配

正式 deck 不少于：

- 1 张封面主视觉。
- 每个大章节至少 1 个章节视觉锚点。
- 每 8-10 页至少 1 个非卡片型视觉页。
- 至少 2 张关键概念图或框架图。
- 教学类 deck 至少 2 张错误/改造对照页或案例拆解页。

## 视觉资产字段

每个资产必须记录：

- `asset_id`：稳定 ID。
- `slides`：使用页码。
- `role`：cover、section、concept、case、background、texture。
- `status`：generated、prompt_ready、editable_graphic、placeholder、not_needed。
- `prompt`：生图提示词或图形实现说明。
- `safe_area`：标题、正文和按钮避让区。
- `fallback`：不能生图时如何替代。
- `risk`：版权、隐私、真实性、可读性风险。

## 禁止项

- 不用无主题关系的抽象背景冒充高级感。
- 不用同一种弧线、光晕、卡片贯穿 30 页以上。
- 不把提示词当成已生成资产。
- 不伪造真实截图、真实品牌、真实客户、真实人物或新闻现场。
- 不把文字烘焙进图片；文字必须保持 HTML/PPT 文本层。

## 混合渲染：局部生图资产

逼近 NotebookLM 整页绘制质感的关键手段：文字和图表骨架留在 HTML 层（可编辑、无错字），把 CSS 画不出来的"有机图形"交给生图，以透明感局部资产嵌入页面。

### 两类资产

1. **隐喻插画**：把该页的视觉隐喻画成一张局部插画（如"曲线冲破关卡""分岔路径"），放在 quote、hero、section、end 等留白较多的页面。
2. **手绘标注**：马克笔质感的箭头、圈选、下划线，叠在图表页上做注释层，是"手工感"的最大来源。

### 提示词配方（已验证）

通用硬要求（每条提示词必须包含）：

- `pure white background`（配合 multiply 融合）。
- `absolutely no text, no letters, no numbers`（图内出现文字必废，中文错字不可接受）。
- `subject centered with generous even margin on all sides`（生成图主体经常偏移，不居中的图嵌入后会像被裁切）。
- 风格锚定：信息图风用 `flat vector line-art, hand-drawn marker feel, black ink + navy (#33506b) + one vermillion red (#d8402c) accent`；纸感风用 `ink line illustration, warm muted tones, editorial book illustration style`。
- 手绘箭头类：`single hand-drawn curved arrow, marker pen feel, slightly rough stroke, nothing else in the image`。

### 嵌入规则（已验证）

- 模板已内置工具类：`<img class="gen-asset" src="assets/gen/xx.png" style="width:210px; left:620px; top:316px;">`。
- `.gen-asset` 自带 `position:absolute + mix-blend-mode:multiply + 提白滤镜`：白底图直接融入白/浅色页面，无需抠图。深色页不能用 multiply，需要真透明 PNG 或改用 `screen` 混合的深底资产。
- 定位用固定舞台坐标（1280×720 的 px 值），优先 `left/top`，图上可直接挂 `data-reveal` 或 `data-step` 参与揭示节奏。
- 需要旋转/镜像时写在 img 的内联 style 上：它会覆盖揭示动画的位移（只淡入不位移），可接受；不可接受时套一层容器 div 做变换。
- **嵌入后必须截图微调**：生成图的主体位置不可控，第一次摆放几乎必然偏，按截图调 left/top/width。
- 每页最多一个生图资产；数据本身仍由 HTML 图表承载，插画只做隐喻和注释，不承载数字。
- 资产存放 `assets/gen/`，命名 `<用途>-<内容>.png`（如 `metaphor-breakthrough.png`、`annotation-arrow-red.png`）。

## HTML 实现要求

- 主视觉要参与页面结构，而不是只做背景。
- 视觉页要有更强的入场方式：景深推进、图层错动、路径揭示、局部放大。
- 内容页可以克制，但每章必须有节奏变化。
- 下载 PPT 后无法保留复杂 HTML 动效时，PPT 至少保留静态主视觉和页面转场。

## 验收判断

交付前问自己：

- 这套 deck 截 3 张图给别人看，是否能看出主题和气质？
- 如果去掉所有颜色，只剩布局，是否仍有层级？
- 视觉资产是否服务概念，而不是只装饰？
- 主视觉、章节视觉和关键概念图是否真实存在，还是只有提示词？
- 如果没有生图，替代方案是否足够高级？
