# 主题内容设计

## 目标

把用户的主题、资料、文章、报告或零散想法收敛成可制作的中文演示结构。先决定“讲给谁、为什么讲、讲完要发生什么”，再决定页数和每页内容。

## 输入检查

优先确认或显式假设：

- 受众：老板、客户、投资人、学生、内部团队、公众用户。
- 场景：汇报、路演、培训、发布会、课程、方案评审。
- 核心行动：讲完后希望观众做什么——批预算、定方案、报名、采纳建议。整套 deck 为它服务，结尾页必须回应它。
- 输出模式：阅读件还是演示件（见下节），决定每页文字密度。
- 时长和页数：没有要求时，默认 8-12 页。
- 素材：文本、数据、图片、链接、会议纪要、品牌资产。
- 约束：保密、引用、品牌调性、是否需要生成 PPT 文件。

Phase 0 风格确认完成后，如果其他信息仍不足，可以基于显式假设继续。主题、受众或目标缺失到会改变整体结构时，只问**恰好一个**问题。风格未确认时服从主 skill 的 Phase 0，不得用“能推断”绕过确认门。

## Design Read（动工前先读题）

大多数糟糕的 AI 演示稿源于跳过读题直接套默认审美。动工前必须先输出一句 Design Read（格式和视觉侧细则见 `13-审美设计与Taste纪律.md`），内容侧至少覆盖：

> 读作：<场景> 面向 <受众>，讲完希望 <核心行动>，用 <气质词> 的语言，选 <风格预设> + <输出模式>。

示例：*"读作：年度业务汇报面向管理层，讲完希望批准预算，用克制的数据叙事语言，选 NotebookLM 信息图风 + 阅读件。"*

判读的依据：用户用过的气质词（"高端""极简""发布会感"）、引用的参考（链接、截图、竞品）、受众身份、既有品牌资产、隐性约束（正式度、保密、行业惯例）。**受众决定审美，不是你的偏好决定审美**。判读能自信推断时不要问，直接声明假设并继续。

## 输出模式：阅读件 vs 演示件

同一主题，两种模式的每页密度完全不同，动工前必须先定：

| | 阅读件（Detailed Deck） | 演示件（Presenter Slides） |
|---|---|---|
| 定位 | 没有演讲者也能读懂 | 演讲者的视觉辅助，不喧宾夺主 |
| 场景 | 邮件发送、会前阅读、结案报告 | 现场演讲、发布会、公开课 |
| 每页文字 | 完整论证：takeaway + 3-5 条要点 + 数据 + 来源 | 一句金句或一个关键数据，正文尽量删 |
| speaker_notes | 可省 | 必须完整（内容都搬到备注里） |

判断规则：用户说"发给别人看""汇报材料""不用我讲"选阅读件；说"我要上台讲""发布会""演讲"选演示件；无法判断时默认阅读件并标注假设。两种模式共用同一套视觉系统和版式库，差别只在信息密度。

## 内容流程

1. 提炼一句主张：整套演示只服务一个核心结论。
2. 建立叙事线：背景 -> 问题 -> 洞察 -> 方案 -> 证据 -> 行动。
3. 切分章节：每章只承担一个任务。
4. 生成逐页规划：每页只表达一个 takeaway。
5. 标注证据状态：已给来源、需补来源、推断、假设。
6. 标注视觉意图：数据图、关系图、流程图、场景图、概念图。
7. 标注动效意图：一次性呈现、分步出现、强调、转场。

常用故事线：

- 问题 -> 原因 -> 方案 -> 证据 -> 行动。
- 背景变化 -> 新机会 -> 能力匹配 -> 路径 -> 结果。
- 现状 -> 差距 -> 原因 -> 改进方案 -> 里程碑。
- 用户痛点 -> 产品方案 -> 价值证明 -> 商业结果 -> 请求。
- 观察 -> 判断 -> 策略 -> 执行 -> 验收。

## 每页字段

- `id`：稳定页码，如 `s01`。
- `type`：cover、agenda、section、content、data、case、process、comparison、summary、closing。
- `title`：短标题，避免空泛。
- `takeaway`：一句话结论。
- `body`：3-5 条要点，避免段落堆叠。
- `evidence`：来源、数据或“需补来源”。来源要具体到可追溯——源文件名+章节、链接或数据表位置，不写"网络资料"这种查不回去的说法。阅读件的关键论证页应在页脚或卡片上放来源行。
- `visual`：图表、图片、版式或图形建议。
- `animation`：页面内动效建议。
- `transition`：进入下一页的 PPT 转场意图。
- `speaker_notes`：讲述提示。

## 文案自检（交付前重读每一条可见文字）

AI 生成的"聪明文案"比平庸文案更伤专业度。交付前逐条重读标题、takeaway、要点、图表标签、页脚，命中以下任何一条就重写：

- **空心动词与万能词**："赋能、助力、引领、打造、极致、全方位、一站式、新范式"。换成具体动作和具体对象——"把审批周期从 14 天压到 3 天"胜过"全方位提升效率"。
- **假精确数字**：没有来源却写 "92%""4.1 倍"。数字要么有出处，要么明确标注为示例，要么删掉。
- **伪深刻修辞**：强行的比喻、故作谦虚、"做减法的艺术"式的空洞金句。金句必须是从内容里长出来的判断，不是装饰。
- **语域漂移**：同一套 deck 里不要混用技术黑话、营销腔和文艺腔。开工时定一种声音，全程保持。
- **标题无立场**：每页标题是一个判断（"存量稳固，增量空间巨大"），不是一个标签（"现状分析"）。
- **破折号节制**：中文文案不要用破折号和 em-dash 做装饰性停顿，用句号、逗号或冒号重组句子。

## 失败路径

- 资料过少：输出最小结构和问题清单，不补造事实。
- 目标冲突：指出冲突，例如“极简 5 页”和“完整覆盖所有细节”，并给出保守方案。
- 事实不确定：标注“假设”或“需来源”。
- 材料冲突：列出冲突点，采用最保守表述，并标注需要用户确认。
- 页数过多：合并重复页，优先保留结论页和证据页。
- 页数过少：保留完整叙事，不强塞复杂细节。
- 用户只给标题：生成可执行大纲，并列出需要补充的材料。

## 验收点

- 每页只有一个核心 takeaway。
- 章节顺序符合受众理解路径。
- 证据、推断、假设清楚分开。
- 每页都有视觉和动效意图。
- 没有编造引用、数字、案例或图片来源。
