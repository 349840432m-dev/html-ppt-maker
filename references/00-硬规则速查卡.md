# 硬规则速查卡（生成前唯一必读）

本卡是全部深度文档的浓缩。生成任何 deck 前通读本卡；深度细节按末尾索引按需查阅，不要求整读。

## 开工五步

1. **先完成 Phase 0 模板确认**：读取 `14-模板学习与风格选择.md`，实际打开 `assets/style-library/index.html`，从真实模板中给出 3 个候选并暂停；确认后再写 Design Read。只有用户已指定模板或明确跳过时才能直接继续。
2. **三旋钮**（记录进 visual-system）：`DESIGN_VARIANCE` / `MOTION_INTENSITY` / `VISUAL_DENSITY`（1-10）。动效声明 = 动效兑现，做不到就调低旋钮。
3. **分开选构图模板与技术预设**：七套构图模板查 `assets/style-library/manifest.json`；两种技术预设查 `08-整体视觉与细节.md`。信息图/战略汇报/NotebookLM → `notebooklm.html`，其他风格根据模板签名走 hand 轨道。金标准成品见 `examples/reference-deck-notebooklm/`。
4. **逐页审美契约**：生成 HTML 前，每页必须填 `layout_intent`、`first_visual_anchor`、`visual_translation`、`element_budget`、`aesthetic_risk`、`pass_criteria`，并跑 `scripts/audit_layout_aesthetics.py`。资料少时也要做视觉对象转换，不能默认落成文字卡片。
5. **按风格选渲染轨道**：NotebookLM 标准页由 `render_deck.py --preset notebooklm` 组装；编辑部纸感风以 `index.html` 走 hand 轨道，禁止混入未适配的 NotebookLM 引擎片段。

## 设计硬规则 Top 10

先用六项基础契约检查整套视觉：**层级清楚、留白主动、默认中性色 + 单强调色、图文各司其职并对齐、数据页只讲一个结论、字体/间距/配色节奏统一**。完整定义见 `13-审美设计与Taste纪律.md`。

1. 每页一个 takeaway，标题是判断不是标签（"存量稳固，增量空间巨大" ✓，"现状分析" ✗）。
2. 选版式先查语义映射表（09）：并列→卡片、对比→contrast、时序→process/gantt、量级→bars、构成→waterfall、断言→quote。
3. 每页一个"第一视觉锚点"，字号为正文 3-5 倍；全页字号挤在 17-24px 是失败页。
4. 每页至少 1 个主视觉结构、1 个辅助解释元素、1 个节奏元素；但主结构只能有 1 个，辅助元素最多 3 个。
5. 红色纪律（信息图风）：每页只标一个最关键信息。单页有彩色 ≤3 种。
6. 一致性守恒：圆角阶梯 ≤2 档（50%/胶囊不计）；边框宽只用带语义的阶梯（如 1px 发丝线 / 2px 卡片框 / 3px 强调 / 4px 坐标轴）；动效时长收敛为一套语义档位（≤6 档）。不得随手加新值。
7. 图表占画布 ≥60%，每个图表页至少一层标注（箭头/图例/来源行）。
8. 数字必须有来源；没有来源写"教学示例"或改定性版式，禁止假精确数字。
9. 连续同类版式 ≤3 页；每 5-7 页一个视觉锚点页。
10. 占位词零容忍：交付前"要素一/说明文字"必须全部替换；结尾页回应核心行动，不是"谢谢观看"。

## Anti-slop Top 10（出现即改）

蓝紫渐变默认套用 ｜ 序号式眉标（"00 / INDEX"）｜ emoji 当图标 ｜ 空心动词（赋能/全方位/一站式）｜ 假精确数字 ｜ 破折号当装饰分隔符 ｜ 三等分卡片无结构关系连排 ｜ 滚动提示/装饰圆点 ｜ 一行 3+ 个中点 `·` ｜ 虚构人物叫"张三/John Doe"

## 技术坑 Top 5

1. 禁用 SVG `<text>`（缩放舞台下错位），图形页文字一律 HTML 绝对定位。
2. 通配选择器 `.slide > *:not(...)` 必须排除所有装饰类，否则装饰层变网格项撑出空白。
3. 带 `data-reveal`/`data-step` 的元素不得靠自身 `transform` 定位或旋转（会被揭示动画覆盖）；居中用 inset+margin，旋转放伪元素。
4. 生图资产用 `.gen-asset` 类嵌入（multiply+提白），嵌入后必须截图微调位置；图内禁一切文字。
5. 禁 `transition: all`、`scale(0)`、UI 入场 `ease-in`；只动 transform/opacity；必须支持 `prefers-reduced-motion`。

## 完成前检查（不可跳过）

本节是浓缩版；全量验收项、命令适用条件和依赖安装命令以 `05-验收清单.md` 为唯一事实源。

> 前置：Codex Desktop 优先加载 bundled workspace dependencies；其他环境按 05 的"建议命令"前置一键安装依赖。脚本自动尝试本机 Chrome，也可显式传 `--chrome`。

1. 先运行 `node scripts/audit_layout_contract.mjs index.html deck-plan.json`，再执行 `layout_probe.js`，清零 DOM 契约、溢出、裁切和重叠 violations；weak-anchor warnings 人工复核。
2. 运行 `scripts/audit_visual_contact_sheet.mjs` 生成全 deck contact sheet：缩略图里主标题可读、首页有主题专属性、连续 4 页不撞轮廓、页面不像线稿占位。
3. 对封面、一个章节页、一个内容页截图：三张图能否看出主题和气质？去色后仍有层级？
4. 跑：`validate_deck_plan.py`、`audit_layout_aesthetics.py`、`audit_layout_contract.mjs`、`audit_style_discipline.py`、`audit_motion_quality.py`，FAIL 必须清零。
5. 逐条重读可见文字做文案自检（01 的清单）；确认成品里已无 `HAND-TRACK` 注释残留。
