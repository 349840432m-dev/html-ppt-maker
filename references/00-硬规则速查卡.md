# 硬规则速查卡（生成前唯一必读）

本卡是全部深度文档的浓缩。生成任何 deck 前通读本卡；深度细节按末尾索引按需查阅，不要求整读。

## 开工五步

0. **先路由任务**：`create/full-redesign` 进入完整流程；`revise` 继承视觉系统；`inspect` 只读；`convert` 先运行 `validate_export_plan.py`，再做运行时审计、导出和验收。旧稿不因缺少新版设计字段被迫改写。
1. **先完成 Phase 0 模板确认**：新建或整套重做时，读取 `14-模板学习与风格选择.md`，实际打开 `assets/style-library/index.html`，从真实模板中给出 3 个候选并暂停；确认前不创建输出目录。只有用户已指定模板或明确跳过时才能直接继续。
2. **确认后再建输出目录**：用户指定路径优先；其次用项目 `outputs/html-ppt-<slug>/`；没有项目时用 `~/Desktop/html-ppt-<slug>/`。deck 产物不得写进 skill 目录。
3. **三旋钮**（记录进 visual-system）：`DESIGN_VARIANCE` / `MOTION_INTENSITY` / `VISUAL_DENSITY`（1-10）。动效声明 = 动效兑现，做不到就调低旋钮。
4. **先选语义骨架，再选主题，最后选技术预设**：三十类内容骨架和七个视觉主题查 `assets/style-library/manifest.json`；两种技术预设查 `08-整体视觉与细节.md`。`selected_template` 是兼容字段，只能写七个主题 ID；NotebookLM / editorial 写入 `visual_system.preset`。金标准成品见 `examples/reference-deck-notebooklm/`。
5. **逐页审美契约**：生成 HTML 前，每页必须填 `layout_intent`、`first_visual_anchor`、`visual_translation`、`element_budget`、`design_contract`、`aesthetic_risk`、`pass_criteria`，并跑 `audit_layout_aesthetics.py`。`design_contract` 包含层级、留白、对齐、强调色和唯一结论。
6. **按风格选渲染轨道**：NotebookLM 标准页由 `render_deck.py --preset notebooklm` 组装；编辑部纸感风以 `index.html` 走 hand 轨道，禁止混入未适配的 NotebookLM 引擎片段。

## 设计硬规则 Top 11

先用六项基础契约检查整套视觉：**层级清楚、留白主动、默认中性色 + 单强调色、图文各司其职并对齐、数据页只讲一个结论、字体/间距/配色节奏统一**。完整定义见 `13-审美设计与Taste纪律.md`。

1. 每页一个 takeaway，标题是判断不是标签（"存量稳固，增量空间巨大" ✓，"现状分析" ✗）。
2. 选版式先查语义映射表（09）：并列→卡片/blueprint、对比改造→contrast/before-after、时序→process/gantt、量级→bars、构成→waterfall/stack、分流→decision-tree、损耗→funnel、机制→causal-chain、闭环→cycle、层级→pyramid/hierarchy、诊断→diagnostic-axis、系统边界→system-map、连续区间→spectrum、核验→checklist、断言→quote。
3. 每页一个"第一视觉锚点"，字号为正文 3-5 倍；全页字号挤在 17-24px 是失败页。
4. 每页至少 1 个主视觉结构、1 个辅助解释元素、1 个节奏元素；但主结构只能有 1 个，辅助元素最多 3 个。
5. 红色纪律（信息图风）：每页只标一个最关键信息。单页有彩色 ≤3 种。
6. 一致性守恒：圆角阶梯 ≤2 档（50%/胶囊不计）；边框宽只用带语义的阶梯（如 1px 发丝线 / 2px 卡片框 / 3px 强调 / 4px 坐标轴）；动效时长收敛为一套语义档位（≤6 档）。不得随手加新值。
7. 图表占画布 ≥60%，每个图表页至少一层标注（箭头/图例/来源行）。
8. 数字必须有来源；没有来源写"教学示例"或改定性版式，禁止假精确数字。
9. 连续同类版式 ≤3 页；每 5-7 页一个视觉锚点页。
10. 占位词零容忍：交付前"要素一/说明文字"必须全部替换；结尾页回应核心行动，不是"谢谢观看"。
11. 图形/图表面板的底色默认与页面背景一致（如纸白 `--bg`），不另起纯白面板；纯白 `--surface` 只留给真正需要卡片层级浮起的内容（如数据卡、档案卡）。同一页不得出现与背景无关的第二种面板底色。

## Anti-slop Top 10（出现即改）

蓝紫渐变默认套用 ｜ 序号式眉标（"00 / INDEX"）｜ emoji 当图标 ｜ 空心动词（赋能/全方位/一站式）｜ 假精确数字 ｜ 破折号当装饰分隔符 ｜ 三等分卡片无结构关系连排 ｜ 滚动提示/装饰圆点 ｜ 一行 3+ 个中点 `·` ｜ 虚构人物叫"张三/John Doe"

## 技术坑 Top 6

1. 禁用 SVG `<text>`（缩放舞台下错位），图形页文字一律 HTML 绝对定位。
2. 通配选择器 `.slide > *:not(...)` 必须排除所有装饰类，否则装饰层变网格项撑出空白。
3. 带 `data-reveal`/`data-step` 的元素不得靠自身 `transform` 定位或旋转（会被揭示动画覆盖）；居中用 inset+margin，旋转放伪元素。
4. 生图资产用 `.gen-asset` 类嵌入（multiply+提白），嵌入后必须截图微调位置；图内禁一切文字。
5. 禁 `transition: all`、`scale(0)`、UI 入场 `ease-in`；只动 transform/opacity；必须支持 `prefers-reduced-motion`。
6. 任何描边/连线不得穿过或压住文字：标签放在图形描边外侧并留 ≥8px 净距（禁止骑线），节点内文字要有实心底衬，连线终止于节点边界。SVG 的 `path/circle/line` 等描边同样受此约束，`layout_probe.js` 的 `stroke-text-overlap` 必须为零；确属有意的描边衬底（如轮廓圆环托在标题下）在元素或祖先上加 `data-stroke-under-text` 豁免，并在验收记录说明。

## 完成前检查（不可跳过）

本节是浓缩版；全量验收项、命令适用条件和依赖安装命令以 `05-验收清单.md` 为唯一事实源。

> 前置：Codex Desktop 优先加载 bundled workspace dependencies；其他环境按 05 的"建议命令"前置一键安装依赖。脚本自动尝试本机 Chrome，也可显式传 `--chrome`。

1. 所有命令通过 skill 目录的 `scripts/run_tool.sh` 调用，不依赖当前工作目录或系统默认 Python/Node。
2. 运行 `audit_layout_contract.mjs` 和 `layout_probe.js`，清零 DOM 契约、溢出、裁切和重叠 violations；weak-anchor warnings 人工复核。
3. 运行 `audit_visual_contact_sheet.mjs` 生成全 deck contact sheet：缩略图里主标题可读、首页有主题专属性、连续 4 页不撞轮廓、页面不像线稿占位。
4. 对封面、一个章节页、一个内容页截图：三张图能否看出主题和气质？去色后仍有层级？
5. 跑：`validate_deck_plan.py`、`audit_layout_aesthetics.py`、`audit_layout_contract.mjs`、`audit_style_discipline.py`、`audit_motion_quality.py`，FAIL 必须清零。
6. 导出 PPTX 后运行 `verify_html_matched_export.py`，核对页数、PNG 尺寸与哈希、PPT 内嵌图片、16:9 画布、分镜差异和转场。
7. 逐条重读可见文字做文案自检（01 的清单）；确认成品里已无 `HAND-TRACK` 注释残留。
