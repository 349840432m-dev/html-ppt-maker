# 资源与脚本索引

本文件是全部 assets、templates、references 和 scripts 的职责说明。需要某类资源时先查这里，不要凭猜测调用脚本。

## 入口与金标准

- `00-硬规则速查卡.md`：开工入口和完成检查；后续 reference 只按当前阶段读取。
- `examples/reference-deck-notebooklm/`：金标准示例成品（NotebookLM 信息图风），生成时用作质量对照。

## Assets

- `assets/style-library/index.html`：一套三十页语义骨架（十二类基础骨架 + 十八类表达骨架）与七个实时视觉主题的交互图谱；Phase 0 必须实际查看，它是预览事实源。
- `assets/style-library/style-gallery.png`：可选的默认主题静态快照；只有在本轮通过渲染脚本重新生成并核验后才能展示，不能代替交互图谱。
- `assets/style-library/manifest.json`：三十个语义 layout、七个主题及图库/回归模式的机器契约；保留 `styles` 键兼容既有 deck-plan。
- `assets/html-deck-template/index.html`：可复制的单文件 HTML 演示模板（编辑部纸感风：衬线标题、米白纸底、深绿强调），自带设计 token 层、固定 16:9 舞台、10 类版式（hero、深色章节页、framework、contrast、data-card、quote、content、case-walkthrough、process-map、结尾页）和分步 reveal / stagger / count-up / 路径绘制动效实现。
- `assets/html-deck-template/notebooklm.html`：第二种风格预设（NotebookLM 信息图风），内置 19 类确定性引擎版式，包括 cycle、radial、pyramid、hierarchy、diagnostic-axis、system-map、spectrum 等关系组件，与 index.html 共用同一套翻页与揭示机制。

## Scripts

- `scripts/run_tool.sh`：统一选择 Codex bundled 或系统 Python/Node，并注入 Node 模块目录；所有命令从 deck 输出目录通过它调用。
- `package.json` / `requirements.txt`：Node 与 Python 直接依赖的固定版本清单；依赖安装在 deck 的 `.runtime/`，不写进 skill 目录。
- `scripts/validate_deck_plan.py`：检查逐页规划字段完整性、manifest 模板 ID、模板签名、技术预设 + 逐版式 `data` 槽位契约。
- `scripts/validate_export_plan.py`：转换既有 HTML 时检查最小兼容契约，只要求非空且 ID 唯一的 slides，不强迫旧稿补齐新版设计字段。
- `scripts/render_deck.py`：双轨组装引擎。读 deck-plan.json，引擎轨道页由片段生成器确定性渲染（含 CSS 按需裁剪），锚点页输出 `HAND-TRACK` 起点存根。
- `scripts/layout_probe.js`：运行时布局探针。在打开成品的浏览器里执行（CDP `Runtime.evaluate` + awaitPromise），逐页检查舞台溢出、文本裁切、元素重叠、连接线穿字（`line-text-overlap`）和 SVG 描边穿字（`stroke-text-overlap`）；`scripts/audit_layout_runtime.py` 是它的 Playwright 驱动（未装 Playwright 时打印手动执行说明）。
- `scripts/audit_deck_quality.py`：检查版式多样性、关键视觉页、动效和转场规划质量。
- `scripts/audit_layout_aesthetics.py`：检查每页是否完成审美设计字段、视觉对象转换、元素预算和五字段 `design_contract`。
- `scripts/audit_layout_contract.mjs`：把 deck-plan 与最终 HTML 逐页对照，检查 slide id、版式 class、第一视觉锚点和三类实际元素角色。
- `scripts/audit_visual_strength.py`：检查高端 deck 是否具备足够视觉资产和生图/替代策略；可附加 `deck-plan.json` 验证视觉资产最低配是否匹配页数、章节和教学属性。
- `scripts/audit_motion_quality.py`：检查 HTML/CSS 是否出现 `transition: all`、`scale(0)`、`ease-in`、缺少 reduced motion 等动效质量问题。
- `scripts/audit_style_discipline.py`：检查典型低质视觉信号，如占位词、默认 AI 紫色渐变、装饰性滚动提示、过度阴影/圆角和破折号滥用。
- `scripts/audit_visual_contact_sheet.mjs`：渲染全部 HTML slides 的最终揭示状态，输出逐页截图、全 deck contact sheet 和 `*-visual-audit.json`，用于人工审稿整套轮廓、首页专属性、可读性、占位感和版式重复。
- `scripts/export_html_matched_pptx.mjs`：渲染 HTML 最终态或 storyboard 状态，生成全页图片 PPTX、截图目录、导出清单并注入页面转场。这是默认 PPT 交付路径。
- `scripts/verify_html_matched_export.py`：核对导出清单、HTML/plan/PPT 页数、截图尺寸和哈希、第 N 页图片关系与第 N 张截图、无裁切/旋转/翻转、16:9 画布、逐页单张全画布图片几何、分镜差异与转场状态；通过后才能标记 HTML-matched。
- `scripts/expand_storyboard_plan.py`：把含 `storyboard_states` 的 deck plan 展开为 PPT 可用的静态状态页 plan。
- `scripts/generate_pptx_from_plan.py`：仅在用户明确要求可编辑文本时生成简化降级版；不得命名或描述为 HTML-matched。
- `scripts/apply_ppt_transitions.py`：对已有 `.pptx` 注入基础页面转场，或检查转场是否存在。
- `scripts/audit_style_library.py`：维护图库时检查 1 个 canonical set、30 个唯一语义页面、7 个实时主题、根主题和兼容字段；普通 deck 生成不必运行。
- `scripts/render_style_gallery.mjs`：默认验证 30 个页面、7 次主题换肤、DOM 身份和筛选/预览，再生成默认主题快照。只有发布前主动传 `--full-matrix` 才验证 210 个渲染状态；全矩阵不是日常必跑项，也不会创建 210 个 DOM 页面。
