# 资源与脚本索引

本文件是全部 assets、templates、references 和 scripts 的职责说明。需要某类资源时先查这里，不要凭猜测调用脚本。

## 入口与金标准

- `00-硬规则速查卡.md`：开工入口和完成检查；后续 reference 只按当前阶段读取。
- `examples/reference-deck-notebooklm/`：金标准示例成品（NotebookLM 信息图风），生成时用作质量对照。

## Assets

- `assets/style-library/index.html`：七种设计语言、每种六类页面骨架的可交互模板图谱；Phase 0 必须实际查看。
- `assets/style-library/style-gallery.png`：模板图谱静态预览，用于在风格确认阶段直接展示。
- `assets/style-library/manifest.json`：七套模板的适用场景、构图签名与禁用项机器契约。
- `assets/html-deck-template/index.html`：可复制的单文件 HTML 演示模板（编辑部纸感风：衬线标题、米白纸底、深绿强调），自带设计 token 层、固定 16:9 舞台、10 类版式（hero、深色章节页、framework、contrast、data-card、quote、content、case-walkthrough、process-map、结尾页）和分步 reveal / stagger / count-up / 路径绘制动效实现。
- `assets/html-deck-template/notebooklm.html`：第二种风格预设（NotebookLM 信息图风：纯白底、上下黑色压边条、超粗黑体标题、黑/藏蓝/朱红三色、2px 黑边线框卡片，每页围绕一个大图表组织），内置基础信息图版式和全部六个高级公开课组件（decision-tree、funnel、causal-chain、matrix、journey、before-after），与 index.html 共用同一套翻页与揭示机制。

## Templates

- `templates/视觉系统模板.md`：用于描述视觉方向。
- `templates/生图提示词模板.md`：用于输出图片生成任务。
- `templates/视觉概念提案模板.md`：用于生成 3 个可比较的视觉方向。
- `templates/视觉资产清单模板.json`：用于记录主视觉、章节视觉和关键概念图。
- `templates/动画与转场规划模板.md`：用于规划 HTML 动效和 PPT 转场。
- `templates/版式谱系模板.md`：用于避免整套 deck 卡片化、同质化。

## Scripts

- `scripts/validate_deck_plan.py`：检查逐页规划字段完整性 + 逐版式 `data` 槽位契约（引擎轨道页的条目数量、数值范围、红色纪律、对应关系等，命中即 FAIL）。
- `scripts/render_deck.py`：双轨组装引擎。读 deck-plan.json，引擎轨道页由片段生成器确定性渲染（含 CSS 按需裁剪），锚点页输出 `HAND-TRACK` 起点存根。
- `scripts/layout_probe.js`：运行时布局探针。在打开成品的浏览器里执行（CDP `Runtime.evaluate` + awaitPromise），逐页检查舞台溢出、文本裁切、元素重叠；`scripts/audit_layout_runtime.py` 是它的 Playwright 驱动（未装 Playwright 时打印手动执行说明）。
- `scripts/audit_deck_quality.py`：检查版式多样性、关键视觉页、动效和转场规划质量。
- `scripts/audit_layout_aesthetics.py`：检查每页是否完成审美设计字段、视觉对象转换和元素预算，防止页面只有标题、文字块和简单线框。
- `scripts/audit_layout_contract.mjs`：把 deck-plan 与最终 HTML 逐页对照，检查 slide id、版式 class、第一视觉锚点和三类实际元素角色。
- `scripts/audit_visual_strength.py`：检查高端 deck 是否具备足够视觉资产和生图/替代策略；可附加 `deck-plan.json` 验证视觉资产最低配是否匹配页数、章节和教学属性。
- `scripts/audit_motion_quality.py`：检查 HTML/CSS 是否出现 `transition: all`、`scale(0)`、`ease-in`、缺少 reduced motion 等动效质量问题。
- `scripts/audit_style_discipline.py`：检查典型低质视觉信号，如占位词、默认 AI 紫色渐变、装饰性滚动提示、过度阴影/圆角和破折号滥用。
- `scripts/audit_visual_contact_sheet.mjs`：渲染全部 HTML slides 的最终揭示状态，输出逐页截图、全 deck contact sheet 和 `*-visual-audit.json`，用于人工审稿整套轮廓、首页专属性、可读性、占位感和版式重复。
- `scripts/export_html_matched_pptx.mjs`：渲染 HTML 最终态或 storyboard 状态，生成全页图片 PPTX、截图目录、导出清单并注入页面转场。这是默认 PPT 交付路径。
- `scripts/expand_storyboard_plan.py`：把含 `storyboard_states` 的 deck plan 展开为 PPT 可用的静态状态页 plan。
- `scripts/generate_pptx_from_plan.py`：仅在用户明确要求可编辑文本时生成简化降级版；不得命名或描述为 HTML-matched。
- `scripts/apply_ppt_transitions.py`：对已有 `.pptx` 注入基础页面转场，或检查转场是否存在。
- `scripts/audit_style_library.py`：维护模板库时检查七种风格、六类页面骨架和 42 个模板页是否齐全；普通 deck 生成不必运行。
