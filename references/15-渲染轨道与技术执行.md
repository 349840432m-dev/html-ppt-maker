# 渲染轨道与技术执行

本文件是工作流"生成 HTML"一步的执行细则：先按已确认风格选择渲染轨道，再遵守硬性技术约束。

## 模板与预设的分工

- **先区分语义骨架、视觉主题和渲染预设**：`assets/style-library/` 的三十类 layout 负责内容关系，七个 theme 负责视觉语言；`assets/html-deck-template/` 的两个 preset 负责技术起点。选定骨架和主题后，把字阶、第一视觉锚点和节奏迁移到技术预设，禁止只换配色。
- 生成 deck 必须以其中一个技术预设为起点；用户提到"NotebookLM""信息图风""图表型 PPT"时选 `notebooklm.html`，其余默认 `index.html`。

## 轨道选择

- **NotebookLM 预设走双轨引擎**：bars、funnel、gantt、checklist、waterfall、stack、decision-tree、causal-chain、matrix、journey、before-after、blueprint、cycle、radial、pyramid、hierarchy、diagnostic-axis、system-map、spectrum 这些结构固定的版式标 `render_track: "engine"`，按版式契约填 `data` 槽位，然后通过 `scripts/run_tool.sh` 调用 `render_deck.py deck-plan.json index.html --preset notebooklm`。不要手写引擎页。
- **编辑部纸感风走 hand 轨道**：以 `assets/html-deck-template/index.html` 为起点手工实现对应版式。当前确定性引擎不支持编辑部 CSS；不要把 NotebookLM 图表片段塞进编辑部模板。引擎支持的语义若走 hand，写 `hand_reason: "editorial preset requires bespoke layout"`。
- **其他视觉主题走 hand 轨道**：瑞士编辑、包豪斯、工业目录、原研哉留白、开发者粗野和计算动态若不能由现有预设原样承载，仍以最接近的技术模板起步，但必须重写视觉 token 和关键版式；`hand_reason` 写 `selected style-library theme requires bespoke composition`。计算动态只用于封面/章节等少数锚点页，正文回到确认的主主题。
- **锚点页走生成**：hero 封面、quote、end、视觉隐喻页标 `render_track: "hand"`；引擎会输出带 `HAND-TRACK` 注释的起点存根，在存根上手工完成设计（读 `09-版式设计指南.md` 对应小节和 `03-HTML动态演示.md` 技术规范）。引擎已支持的版式（bars、gantt 等）原则上不走 hand 轨道；确有特殊设计需求时必须在该页写 `hand_reason` 字段说明理由，否则校验器会 WARN——不要用 hand 轨道绕过槽位契约。

## 硬性技术约束

- **数字字体单独建制**：视觉系统必须定义 `--font-number`，大数字、年份、序号、百分比和指标单位不得直接继承中文衬线标题字体。纯数字节点使用 lining/tabular numerals；数字与中文单位必须包在同一个布局容器中，以基线或底边对齐。标题中的大数字需要独立 `<span>`，不能让 `50+`、`486` 等数字被宋体或 Georgia 接管。
- **关系线与文字分层**：时间线、迁移图、道路图、因果链的线条只能位于专用轨道，连接线终点停在节点边界或标签前。禁止用绝对定位斜线直接穿越文字区；优先用 Grid/Flex 建立文字层、节点层和线路层，再用短连接段表达关系。承载语义关系的线、点和箭头不得用 `::before` / `::after` 绘制，因为伪元素无法被 DOM 碰撞审计可靠枚举；必须使用真实元素并统一加 `data-connector`。
- **图片先声明用途再适配比例**：产品截图、数据图、界面和实物图标记 `data-image-role="inspect"`，观众必须能看清核心内容；氛围图才允许自由 `cover`。原图与容器比例不同，不得在 `cover` 和 `contain` 之间机械二选一：优先重排网格，其次使用内容保护型裁切。经过人工确认的裁切加 `data-crop-approved`，否则运行时审计会报告 `aggressive-image-crop` 或 `inspect-image-cropped`。
