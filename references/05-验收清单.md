# 验收清单

任何一项无法诚实打勾，deck 就没有完成；修完再交付，不要注明"已知问题"了事。

快捷入口：硬规则浓缩版见 `00-硬规则速查卡.md`；质量对照物见 `examples/reference-deck-notebooklm/`——交付前把自己的封面、内容页、结尾页与它并排比一次。

## 内容

- 已明确受众、目标、场景、核心行动、输出模式（阅读件/演示件）和页数。
- 已完成文案自检（见 `01-主题内容设计.md`）：无空心动词、无假精确数字、无伪深刻修辞、语域统一、每页标题有立场。
- 每页有标题、takeaway、正文、视觉建议、动效建议、备注。
- 每页密度与输出模式一致：阅读件无演讲者也能读懂；演示件每页只留金句或关键数据，内容进 speaker_notes。
- 证据、假设、推断分开标注。
- 关键论证页的来源可追溯（源文件+位置或链接），阅读件的来源行已放在页面上。
- 结尾页回应了核心行动，不是"谢谢观看"。
- 没有伪造来源、数据、图片、截图或品牌素材。

## 视觉与生图

- 已写 Design Read，且与受众、场景、目标一致。
- 已设置 `DESIGN_VARIANCE`、`MOTION_INTENSITY`、`VISUAL_DENSITY`，并能解释它们如何影响版式、动效和密度。
- 成品与旋钮一致：声明了高 `MOTION_INTENSITY` 页面就真的在动，做不到就调低旋钮交付干净的静态版（动效声明 = 动效兑现）。
- 已完成 CRAP 检查：对比、重复、对齐、亲密性成立。
- 已完成六项基础设计检查：三级信息层级清楚；留白有意图；默认中性色 + 单强调色；图文共享对齐线且各司其职；每个数据页只证明一个结论；全篇 token 与节奏统一。
- 有完整视觉系统。
- 高端视觉任务有 3 个视觉概念方向和 1 个选定方向。
- 有视觉资产清单，至少覆盖封面、章节、关键概念图。
- 有版式谱系，长 deck 至少 6 类版式。
- 公开课、培训课或 NotebookLM 风 deck 至少使用 3 类高级图表组件（decision-tree、funnel、causal-chain、matrix、journey、before-after）。
- 每页已完成逐页版式审计：有版式意图、第一视觉锚点、视觉对象转换、元素预算、审美风险和通过标准。
- 逐页版式审计已写明 `hierarchy`、`whitespace_intent`、`alignment_contract`、`color_emphasis`、`single_claim`；不接受“简洁、高级、留白多”这类不可验证描述。
- 关键概念页有视觉隐喻（概念翻译成可画的图像），不是全部落成要点卡片。
- 每 5-7 页有明显节奏变化或视觉锚点。
- 生图页面、非生图页面、需用户提供素材页面已区分。
- 产品截图、数据图、界面和实物图已标记 `data-image-role="inspect"`；关键文字、数值、控件和主体在演示距离下可辨认。
- 图片比例与容器比例经过审计；`inspect-image-cropped` 为零，`aggressive-image-crop` 已处理。使用 `data-crop-approved` 的页面有人工审稿依据，不是为了绕过检查。
- 生图提示词包含主体、风格、比例、构图、禁用项和文字避让。
- 混合渲染资产（如使用）：图内没有任何文字；白底图经 `.gen-asset` 融合后无灰底框；嵌入位置已按截图微调；每页不超过一个生图资产；数据仍由 HTML 图表承载。
- 字体、颜色、图片不会造成内容不可读。
- 所有大数字、年份、序号和指标使用独立数字字体链；数字与中文单位组合为一个视觉单元，没有宋体/Georgia 数字、基线错位或单位掉行。
- 如果用户要求“高端”“有审美”“公开课”“路演”，必须完成自评，低于 4 分不得交付。
- 如果用户要求“高端”“大气”“有审美”，必须运行视觉资产审计。
- 每个普通内容页至少有 1 个主视觉结构、1 个辅助解释元素、1 个节奏元素；主结构只能 1 个，辅助元素最多 3 个，装饰元素最多 1 个且必须有信息作用。
- 没有默认 AI 紫蓝渐变、三等分卡片、装饰性标签、无意义状态条、破折号滥用或泛标签。

## HTML

- `index.html` 存在。
- 双轨渲染（若适用）：引擎轨道页由 `render_deck.py` 组装且槽位契约校验通过；成品里没有残留 `HAND-TRACK` 注释。
- 运行时布局审计：对全部页面执行 `layout_probe.js`（或 `audit_layout_runtime.py`），溢出、文本裁切、元素重叠 violations 为零。
- 时间线、迁移线、道路、关系节点和箭头均使用真实 DOM 元素并标记 `data-connector`；没有用 `::before` / `::after` 承载语义图形，静态审计的 `pseudo-connector` 为零。
- 运行时审计的 `line-text-overlap` 为零，时间线、迁移线、道路和箭头没有穿过标题、标签、注释或来源行。
- `numeric-font-mismatch` warnings 已逐条处理；确需衬线数字的品牌/历史档案页必须在验收记录说明理由。
- DOM 契约审计：`audit_layout_contract.mjs` 已确认每页 id、版式、第一视觉锚点、主结构、辅助元素和节奏元素真实存在。
- 能在浏览器打开并看到第一页。
- 支持键盘或点击翻页。
- 支持 `B` 键静态最终态。
- 页面内动效克制，且支持减少动画。
- 动效规划使用了标准词汇，如 Reveal、Stagger、Line drawing、Number ticker、Direction-aware transition。
- 每个主要动效都有目的：阅读顺序、空间关系、状态反馈、结构解释或避免突兀切换。
- 没有 `transition: all`、`scale(0)` 或 UI 入场 `ease-in`。
- Stagger 间隔在 30-80ms；高频键盘翻页不被长动画阻塞。
- hover 动效已限制在支持 hover 的精细指针设备。
- PPT 降级方式已写清：保留静态最终态、拆页、或使用页面转场。
- 文本没有明显溢出或重叠（以 `layout_probe.js` 的结果为准，不靠目测）。
- 已生成全 deck contact sheet，并人工审稿：缩略图中主标题可读、首页有主题专属性、连续页面轮廓不重复、页面不像线稿占位，关键视觉页足够撑起气质；另单独放大检查所有大数字页、关系图页和检查型图片页。
- 已完成渲染自检：对封面、至少一个章节页、至少一个内容页截图，三张图能看出主题和气质；去掉颜色只看布局仍有层级。
- contact sheet 人工审稿时已额外检查：是否有像漏放内容的空白页、同权重文字页、多强调色争抢页、图文各自漂浮页、同页多个数据结论。
- 整套 deck 至少出现 3 种入场方式（如分层入场、路径绘制、count-up、对照揭示），不是所有页面都是"卡片逐个淡入"。
- 没有默认套用科技蓝紫渐变。

## PPT

- 如要求 `.pptx`，文件存在且非空。
- 默认交付文件由 `export_html_matched_pptx.mjs` 从 HTML 截图生成；简化可编辑版只能作为明确降级文件。
- 页数与规划一致。
- HTML 匹配版导出后已检查 `*-export.json`：HTML 页数、导出页数、截图数和 transition 状态一致；分镜模式允许导出页数多于 HTML 页数。
- 转场策略已应用或明确降级。
- HTML 动效无法进入 PPT 的部分已说明。
- 关键 reveal 页如需保留讲授节奏，已补 `storyboard_states` 并生成 `storyboard-plan.json`。

## 建议命令

前置：Codex Desktop 先加载 bundled workspace dependencies；其他环境在 skill 目录执行：

```bash
npm install playwright sharp pptxgenjs        # Node 侧：contact sheet 与 PPTX 导出
pip3 install playwright                        # Python 侧：运行时布局审计
python3 -m playwright install chromium         # 已装本机 Chrome 可跳过
```

脚本会自动尝试本机 Google Chrome，也可用 `--chrome` 显式指定浏览器路径。

```bash
python3 scripts/validate_deck_plan.py deck-plan.json
python3 scripts/render_deck.py deck-plan.json index.html --preset notebooklm
python3 scripts/audit_deck_quality.py deck-plan.json
python3 scripts/audit_layout_aesthetics.py deck-plan.json
node scripts/audit_layout_contract.mjs index.html deck-plan.json --out layout-contract.json
python3 scripts/audit_visual_strength.py visual-concepts.json
python3 scripts/audit_style_discipline.py index.html deck-plan.json
python3 scripts/audit_motion_quality.py index.html
node scripts/audit_visual_contact_sheet.mjs --out visual-audit index.html
python3 scripts/audit_layout_runtime.py index.html         # 或在浏览器里执行 layout_probe.js
python3 scripts/expand_storyboard_plan.py deck-plan.json storyboard-plan.json
node scripts/export_html_matched_pptx.mjs index.html deck-html-matched-with-transitions.pptx --plan deck-plan.json
python3 scripts/apply_ppt_transitions.py --check deck.pptx
python3 scripts/audit_style_library.py .    # 仅在维护 skill 模板库时运行，普通 deck 生成不必
```
