# 模板学习与风格选择

## 目的

风格确认必须基于真实构图预览，而不是只给风格名称和形容词。`assets/style-library/index.html` 只有一套三十页语义骨架，七个视觉主题作用于同一份 DOM：基础骨架回答“这一页承担什么叙事角色”，表达骨架回答“内容之间是什么关系”，主题回答“同一内容以什么视觉语言呈现”。

## 使用顺序

1. 读取 `assets/style-library/manifest.json`，先按故事关系确定需要的 layout，再按受众、场景和内容类型筛出 3 个主题候选。
2. 打开 `assets/style-library/index.html`，先用 LAYOUT 检查封面、目录、章节、观点、图文、数据、对照、时间线、矩阵、案例、流程、收束十二类基础骨架，再检查决策树、漏斗、因果链、旅程、循环、放射、金字塔、层级树、条形、瀑布、甘特、清单、前后改造、堆叠、蓝图、诊断坐标、系统图和光谱十八类表达骨架是否能承载当前故事线。
3. 保持同一 layout 不变，用 THEME 切换七个主题，比较字体、色彩、边框、视觉节奏和强调方式；主题不得改变节点数量、文字、结构、阅读顺序或隐藏语义内容。
4. 风格确认卡中写出候选主题 ID、最适合的页面和明显取舍；交互 HTML 是唯一预览事实源，不能用未在本轮生成的旧静态图替代。
5. 用户确认后，把选定主题的 `signatures` 写进 `visual-system.json`，每页至少兑现其中两个视觉签名。
6. 生成后按“所选骨架 + 所选主题”审稿。只换颜色而没有保留字阶、视觉锚点和节奏，视为没有使用该主题。

## 内容关系路由

- 有条件和不同终局：`decision-tree`；只有单一路径推进：`process`。
- 讲阶段损耗：`funnel`；讲项目量级高低：`bars`；讲总量如何由增减项构成：`waterfall`。
- 讲“为什么产生结果”：`causal-chain`；讲用户动作与后台机制并行：`journey`。
- 讲持续回流：`cycle`；讲一个中心由多个平行支柱支撑：`radial`。
- 讲基础到高阶价值：`pyramid`；讲父子从属和责任分解：`hierarchy`。
- 讲持续时间、并行和依赖：`gantt`；讲可执行与可验收状态：`checklist`。

## 七个视觉主题的角色

| 主题 ID | 核心用途 | 不宜作为默认主题的情况 |
|---|---|---|
| `swiss-editorial` | 商业公开课、咨询报告、方法论 | 需要亲切、柔软或强情绪叙事 |
| `magazine-editorial` | 观点、案例、品牌故事 | 连续数据页和操作手册 |
| `bauhaus-geometric` | 封面、章节、概念关系 | 严肃企业汇报和密集正文 |
| `dieter-rams` | 产品、功能、参数、技术方案 | 纯观点演讲和娱乐主题 |
| `kenya-hara` | 高端封面、金句、转折页 | 公开课全套或信息密集型 deck |
| `brutalist-developer` | AI、代码、开发者主题 | 消费品牌和高端生活方式 |
| `field-motion` | 动态封面、发布页、章节转场 | 长文、数据和流程页 |

## 混合规则

- 一套 deck 只设一个主主题，可选一个辅主题用于封面或章节页。
- 主主题负责正文、数据和流程；辅主题最多覆盖 20% 页面。
- 推荐组合：`swiss-editorial + bauhaus-geometric`、`magazine-editorial + kenya-hara`、`brutalist-developer + field-motion`、`dieter-rams + swiss-editorial`。
- 禁止把七种风格逐页轮换。那会形成样式拼盘，而不是视觉系统。

## 中文适配

- 中文大标题优先控制在 8–18 字；无法形成英文式极端断行时，用字重、行距和标点建立节奏。
- 衬线标题优先使用系统可用的宋体/明朝体族，正文使用清晰无衬线；不依赖远程字体才能成立。
- 等宽风格中的中文正文不要全部使用等宽字体，只把标签、代码、页码和元数据设为等宽。
- 原研哉留白风不能靠缩小所有文字制造“高级感”；必须保留一个明确的视觉对象。

## 验收

- 风格确认卡包含 3 个真实主题 ID，并说明采用的语义骨架。
- 选定主题为兼容既有 schema 仍写入 `style_confirmation.selected_template`，且必须属于上表七个 ID；NotebookLM / editorial 属于技术预设，写入 `visual_system.preset`。
- `visual-system.json` 记录 `template_signatures` 和 `secondary_template`（没有则为 `null`）。
- 全套 contact sheet 中能辨认主主题的字体、边界和节奏，而不只是一层配色。

## 图库维护与回归

- 正式图库始终只有 30 个 `.slide-template`，每个 `data-layout` 恰好一个；七个主题只切换根节点 `data-theme`。
- 默认回归按改动范围执行：改 layout 时检查该 layout 的七主题；改主题时检查该主题的三十 layout；只改文档时不跑浏览器矩阵。
- `30 × 7 = 210` 只表示可选的发布前全矩阵渲染状态。它不是正式图库页数，也不是每次修改都必须执行的固定成本。
