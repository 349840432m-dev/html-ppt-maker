# 高级图表页组件

## 何时读取

当用户要求公开课、培训课、路演、产品方法论、运营方法论、NotebookLM 风、信息图风或"更高级"时读取。目标是把普通"标题 + 卡片"页面改成有判断、有结构、有图解的讲授页面。

## 组件选择原则

- 一页只讲一个判断，图表负责证明判断。
- 优先选择能表达关系的图形：路径、因果、矩阵、漏斗、决策树、前后对照。
- 能用长度、位置、层级、箭头表达的信息，不要退回普通列表。
- 关键页必须有标注层：箭头、红线、圈注、注释、图例或来源行。
- 红色只标一个决策点、风险点或突破口。

## 新增组件

### 决策树页 decision-tree

- **用途**：把"是否应该做某动作"拆成条件判断。适合付费问卷、产品上线、激励策略、渠道选择。
- **结构**：左侧起点问题 → 2-3 层条件分叉 → 右侧结论标签。
- **设计要点**：条件节点用黑框白底；通过路径用藏蓝，风险/停止路径用红色；终点必须是可执行动作，不是抽象口号。
- **动效**：起点 Reveal → 主路径 Line drawing → 条件节点 Stagger → 终点结论 Pop in。
- **常见错误**：分叉太多；每个结论都像"建议优化"这种空话。

### 漏斗诊断页 funnel

- **用途**：表达从曝光、开始答卷、完成答卷、支付/领取奖励等逐层流失。
- **结构**：4-5 层横向漏斗 + 每层转化率/流失原因 + 右侧红色瓶颈标注。
- **设计要点**：每层宽度对应相对量级；瓶颈层用红色边或红点，不整页变红。
- **动效**：层级从上到下 Stagger，瓶颈标注最后 Reveal。
- **常见错误**：没有量级差异；漏斗只是装饰三角形。

### 因果链页 causal-chain

- **用途**：说明一个设计动作如何影响用户心理、行为和业务结果。
- **结构**：动作 → 感知 → 行为 → 数据结果，4-5 个节点横向推进。
- **设计要点**：每个箭头上写机制，不只写"导致"；结果节点用更强视觉权重。
- **动效**：节点和箭头同步推进，最后高亮结果。
- **常见错误**：只列步骤，没有解释中间机制。

### 矩阵判断页 matrix

- **用途**：比较方案优先级、奖励类型、用户分层、题目价值。
- **结构**：2x2 或 3x3 坐标矩阵 + 象限命名 + 红色目标区。
- **设计要点**：坐标轴必须有明确含义；目标区用浅红底/红框，散点或方案标签放进象限。
- **动效**：轴线绘制 → 象限标签 → 点位 Stagger → 目标区 Highlight。
- **常见错误**：坐标轴含义虚；点位没有理由。

### 用户路径蓝图片 journey

- **用途**：公开课中拆解完整流程，如"看到问卷 -> 决定是否答 -> 答题 -> 领取奖励 -> 复访"。
- **结构**：横向阶段泳道，上层用户动作，下层后台/产品机制，中间标注风险点。
- **设计要点**：阶段数 4-6 个；风险点用红色圆点，产品机制用藏蓝线框。
- **动效**：阶段线推进，风险点逐个浮现。
- **常见错误**：只画用户步骤，没有后台机制。

### 前后对照滑块页 before-after

- **用途**：展示错误问卷设计与改造后方案。
- **结构**：左侧 before、右侧 after，中间可用垂直分割线或滑块隐喻。
- **设计要点**：两边内容必须一一对应；after 不能只是更好看，必须指出改造原则。
- **动效**：先出现 before，再用 Clip-path/Mask Reveal after。
- **PPT 降级**：拆成 before 页、after 页和对照总结页。
- **常见错误**：before/after 不对仗；只做视觉差异，没有方法论差异。

### 循环闭环页 cycle

- **用途**：增长飞轮、持续学习、迭代治理等真正回到下一轮输入的机制。
- **结构**：中心目标 + 3–6 个环形阶段 + 明确闭合环线。
- **常见错误**：把普通步骤强行画成飞轮；阶段只有名词，没有写清输出。

### 中心放射页 radial

- **用途**：战略支柱、产品能力、生态角色等“一个中心、多维支撑”的关系。
- **结构**：中心命题 + 3–6 根射线 + 外围节点；只允许一个 `hot` 重点。
- **常见错误**：外围分支互相包含；超过 6 个节点仍挤在一页。

### 价值金字塔页 pyramid

- **用途**：能力成熟度、需求层级、基础到价值的递进。
- **结构**：3–5 层，由上到下逐渐变宽；每层写标题和承担的作用。
- **常见错误**：只有大小差异，没有层级依赖；用面积伪装精确数值。

### 组织层级页 hierarchy

- **用途**：目标分解、组织责任、系统模块和课程知识树。
- **结构**：一个根节点 + 2–4 个分组 + 每组 1–3 个子项。
- **常见错误**：把跨组流程塞进层级树；分组或子项没有责任边界。

### 诊断坐标页 diagnostic-axis

- **用途**：用两个连续变量定位问题、机会或方案，如影响 × 证据、可靠性 × 自治程度。
- **结构**：带名称与数值范围的 x/y 轴 + 一个目标区 + 3–8 个点位。
- **数据契约**：`x_axis`、`y_axis` 含 `label/min/max`；`target` 含 `label/x_min/x_max/y_min/y_max`；`points` 每项含 `label/x/y`，最多一个 `hot`。
- **设计要点**：点位必须位于坐标内侧 90% 安全区；目标区承担第一视觉锚点；轴线使用真实 `[data-connector]`。
- **常见错误**：没有数值尺度；点位标签互相遮挡；把普通 2×2 象限误写成连续诊断轴。

### 系统边界页 system-map

- **用途**：解释信号如何穿过核心模块成为输出，适合产品架构、AI 工作流和治理边界。
- **结构**：1–3 个输入 → 2–4 个核心模块 → 1–3 个输出，外层以 `boundary` 命名系统。
- **数据契约**：`boundary` 字符串；`inputs`、`outputs` 为字符串数组；`modules` 每项含 `title/detail`，最多一个 `hot`。
- **设计要点**：只表现一条主要流向；模块边界与关系线分层；连接线必须是实际 DOM。
- **常见错误**：把组织树或自由散点塞进系统图；没有输入输出；所有模块都高亮。

### 连续光谱页 spectrum

- **用途**：表达成熟度、风险、控制强度、自治程度等从一端到另一端的连续区间。
- **结构**：互斥的左右端点 + 3–6 个停止点 + 一个当前 marker，可选目标区间。
- **数据契约**：`left_label`、`right_label`；`stops`；`marker.label/detail/position_pct`（5–95）；可选 `target.label/start_pct/end_pct`。
- **设计要点**：marker 是第一视觉锚点；目标区只标推荐范围；端点标签必须保持舞台内安全边距。
- **常见错误**：端点含义不互斥；marker 靠边溢出；把离散类别强行排成连续光谱。

## 公开课默认配比

35 页公开课建议至少包含：

- 2-3 页决策树。
- 2 页漏斗诊断。
- 3 页因果链。
- 2 页矩阵判断。
- 2 页用户路径蓝图。
- 2 页前后对照。
- 1–2 页诊断坐标或系统图。
- 1 页连续光谱，用于明确成熟度、风险或自治边界。
- 2–4 页按内容选择 cycle、radial、pyramid 或 hierarchy；只在关系真实成立时使用。
- 每 5-7 页至少一个上述高级组件，避免卡片页连续堆叠。

## HTML 实现要求

- NotebookLM 预设优先由 `render_deck.py --preset notebooklm` 生成，不手工重写引擎已支持的组件。
- 组件标识统一使用 `journey`；兼容读取旧值 `journey-blueprint`，输出一律写 `layout-journey`。
- 节点文字用 HTML，线条和箭头用 SVG 或 CSS border；不要用 SVG `<text>`。
- 所有组件必须支持 `data-step`，便于分镜式 PPT 导出。
- PPT 最终态必须和 HTML 最终态一致。
