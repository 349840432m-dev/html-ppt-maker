# 高级图表页组件

## 何时读取

当用户要求公开课、培训课、路演、产品方法论、运营方法论、NotebookLM 风、信息图风或"更高级"时读取。目标是把普通"标题 + 卡片"页面改成有判断、有结构、有图解的讲授页面。

## 组件选择原则

- 一页只讲一个判断，图表负责证明判断。
- 优先选择能表达关系的图形：路径、因果、矩阵、漏斗、决策树、前后对照。
- 能用长度、位置、层级、箭头表达的信息，不要退回普通列表。
- 关键页必须有标注层：箭头、红线、圈注、注释、图例或来源行。
- 红色只标一个决策点、风险点或突破口。

## 新增组件

### 决策树页 decision-tree

- **用途**：把"是否应该做某动作"拆成条件判断。适合付费问卷、产品上线、激励策略、渠道选择。
- **结构**：左侧起点问题 → 2-3 层条件分叉 → 右侧结论标签。
- **设计要点**：条件节点用黑框白底；通过路径用藏蓝，风险/停止路径用红色；终点必须是可执行动作，不是抽象口号。
- **动效**：起点 Reveal → 主路径 Line drawing → 条件节点 Stagger → 终点结论 Pop in。
- **常见错误**：分叉太多；每个结论都像"建议优化"这种空话。

### 漏斗诊断页 funnel

- **用途**：表达从曝光、开始答卷、完成答卷、支付/领取奖励等逐层流失。
- **结构**：4-5 层横向漏斗 + 每层转化率/流失原因 + 右侧红色瓶颈标注。
- **设计要点**：每层宽度对应相对量级；瓶颈层用红色边或红点，不整页变红。
- **动效**：层级从上到下 Stagger，瓶颈标注最后 Reveal。
- **常见错误**：没有量级差异；漏斗只是装饰三角形。

### 因果链页 causal-chain

- **用途**：说明一个设计动作如何影响用户心理、行为和业务结果。
- **结构**：动作 → 感知 → 行为 → 数据结果，4-5 个节点横向推进。
- **设计要点**：每个箭头上写机制，不只写"导致"；结果节点用更强视觉权重。
- **动效**：节点和箭头同步推进，最后高亮结果。
- **常见错误**：只列步骤，没有解释中间机制。

### 矩阵判断页 matrix

- **用途**：比较方案优先级、奖励类型、用户分层、题目价值。
- **结构**：2x2 或 3x3 坐标矩阵 + 象限命名 + 红色目标区。
- **设计要点**：坐标轴必须有明确含义；目标区用浅红底/红框，散点或方案标签放进象限。
- **动效**：轴线绘制 → 象限标签 → 点位 Stagger → 目标区 Highlight。
- **常见错误**：坐标轴含义虚；点位没有理由。

### 用户路径蓝图片 journey

- **用途**：公开课中拆解完整流程，如"看到问卷 -> 决定是否答 -> 答题 -> 领取奖励 -> 复访"。
- **结构**：横向阶段泳道，上层用户动作，下层后台/产品机制，中间标注风险点。
- **设计要点**：阶段数 4-6 个；风险点用红色圆点，产品机制用藏蓝线框。
- **动效**：阶段线推进，风险点逐个浮现。
- **常见错误**：只画用户步骤，没有后台机制。

### 前后对照滑块页 before-after

- **用途**：展示错误问卷设计与改造后方案。
- **结构**：左侧 before、右侧 after，中间可用垂直分割线或滑块隐喻。
- **设计要点**：两边内容必须一一对应；after 不能只是更好看，必须指出改造原则。
- **动效**：先出现 before，再用 Clip-path/Mask Reveal after。
- **PPT 降级**：拆成 before 页、after 页和对照总结页。
- **常见错误**：before/after 不对仗；只做视觉差异，没有方法论差异。

## 公开课默认配比

35 页公开课建议至少包含：

- 2-3 页决策树。
- 2 页漏斗诊断。
- 3 页因果链。
- 2 页矩阵判断。
- 2 页用户路径蓝图。
- 2 页前后对照。
- 每 5-7 页至少一个上述高级组件，避免卡片页连续堆叠。

## HTML 实现要求

- NotebookLM 预设优先由 `render_deck.py --preset notebooklm` 生成，不手工重写引擎已支持的组件。
- 组件标识统一使用 `journey`；兼容读取旧值 `journey-blueprint`，输出一律写 `layout-journey`。
- 节点文字用 HTML，线条和箭头用 SVG 或 CSS border；不要用 SVG `<text>`。
- 所有组件必须支持 `data-step`，便于分镜式 PPT 导出。
- PPT 最终态必须和 HTML 最终态一致。
