# 整体视觉与细节

本文档定义两个技术渲染预设的视觉基因和全局细节。七套构图参考模板与选择规则见 `14-模板学习与风格选择.md`；逐版式规范见 `09-版式设计指南.md`。

## 两层路由：构图模板 + 技术预设

先从模板学习库确定构图语言，再选择最接近的技术预设。构图模板决定网格、字阶、视觉锚点和节奏；技术预设只决定已有 CSS、交互和确定性渲染能力。两者不能混为一谈。

| 用户词 / 场景 | 首选构图模板 | 技术预设 / 轨道 | 审美硬边界 |
|---|---|---|---|
| NotebookLM、信息图、战略汇报、数据论证 | NotebookLM 信息图（既有金标准） | `notebooklm.html` / engine + hand | 黑、藏蓝、朱红、灰；图表占主体 |
| 商务、咨询、方法论、专业感 | `swiss-editorial` | `notebooklm.html` 严格瑞士化或 hand | 严格网格、直角、发丝线、单强调色 |
| 课程、公开课、观点、品牌叙事 | `magazine-editorial` | `index.html` / hand | 衬线标题、杂志栏位、图片或引语参与结构 |
| 概念发布、章节页、文化创意 | `bauhaus-geometric` | `index.html` / hand | 几何体承担关系，不做装饰性形状堆叠 |
| 产品、功能、参数、技术方案 | `dieter-rams` | `index.html` / hand | 单一产品主体、规格标注、功能性色彩 |
| 高端产品、品牌理念、金句转折 | `kenya-hara` | `index.html` / hand | 70% 以上留白但必须有单一视觉对象 |
| AI、代码、开发者、架构 | `brutalist-developer` | `notebooklm.html` 为骨架 / hand | 等宽标签、原生表格、终端语法服务解释 |
| AI 发布、动态封面、章节转场 | `field-motion` 作为辅模板 | `index.html` / hand | 全屏运动只用于锚点页，正文使用稳定主模板 |

## 两个技术预设

| 预设 | 文件 | 气质 | 适用 |
|---|---|---|---|
| 编辑部纸感风（默认） | `assets/html-deck-template/index.html` | 杂志、书籍、正式演讲 | 课程、路演、品牌叙事、人文主题 |
| NotebookLM 信息图风 | `assets/html-deck-template/notebooklm.html` | 咨询报告、战略文档、数据叙事 | 年度规划、业务汇报、数据驱动的论证 |

选择规则：用户提到"NotebookLM""信息图""图表型""战略汇报"用后者；其余默认前者。技术预设不可逐页混用，但可以依据已确认的构图模板重写其视觉 token 和关键版式。

Design Read 和三个旋钮（`DESIGN_VARIANCE` / `MOTION_INTENSITY` / `VISUAL_DENSITY`）的定义、默认值表和落地规则见 `13-审美设计与Taste纪律.md`；本文档的所有尺寸和密度规范在旋钮取默认值时适用，旋钮偏离默认时按 13 的落地规则整体缩放。

生成前还必须读取 `13-审美设计与Taste纪律.md`，写出 Design Read，并设置 `DESIGN_VARIANCE`、`MOTION_INTENSITY`、`VISUAL_DENSITY`。这三个旋钮决定本套 deck 是偏瑞士网格、电子杂志、信息图密度，还是更接近发布会 hero 叙事。

## 视觉基因：编辑部纸感风

- **色彩**：米白纸底 `#f6f4ee` + 白面板；主文字近黑 `#17191d`，次级 `#4b4f56`，弱化 `#8a8f98`；主强调深绿 `#0e6e60`，对比暖橙 `#b4552d`。深色页用近黑蓝 `#101418` + 米白文字。
- **字体**：标题衬线（`Noto Serif SC` → 宋体族降级），字重 600；正文无衬线（PingFang SC 族）；数字 `tabular-nums`。
- **字号阶梯**（1280×720 舞台）：h1 76px / h2 44px / takeaway 22px / 正文 17-21px / meta 13px。
- **质感**：深色页必须有颗粒噪点纹理 + 双色 radial 光晕（绿主橙辅），避免平涂；浅色页靠纸色和细边框（`rgba(23,25,29,.12)`）分层。
- **圆角**：卡片 8-10px，克制使用。

## 视觉基因：NotebookLM 信息图风

- **色彩**：纯白底 `#ffffff`；只有四个颜色——黑 `#141414`、藏蓝 `#33506b`、朱红 `#d8402c`、灰 `#b9bdc2`。没有渐变、没有浅色容器滥用。
- **红色纪律（最重要的规则）**：红色每页只标**一个**最关键的信息——红线、突破口、决策点、目标值。红色一多，整页的判断力就消失。
- **字体**：全部无衬线；标题字重 900（超粗），正文 600-700。不用衬线。
- **字号阶梯**：h1 64-72px / h2 40px / 图表标签 15-17px / 图表数值 18-21px 加粗 / meta 13px。
- **线框**：卡片和表格用 `2px solid` 黑色实线边框，不用阴影不用圆角（清单行例外，勾选块圆角 10px）。
- **签名元素**：页面上下各一条 10px 黑色压边条（用 `.stage::before/::after` 实现，不占布局）。

## 排版与网格（两预设通用）

- **行高**：标题 1.14-1.2；正文 1.5-1.6；图表标签 1.35-1.4。行高松一档，整页就散。
- **字距**：中文大标题保持 0~+0.01em，**不要模仿英文排版用负字距**（汉字会粘连）；kicker/小标签用 0.18-0.35em 拉开字距制造"刻字感"。
- **页边距**：纸感风 `64px 72px 48px`，信息图风 `52px 64px 40px`（黑色压边条内侧）。全部内容落在这个安全框内，页脚 meta 行的位置全 deck 固定不动——观众感知不到网格，但感知得到"没网格"。
- **对齐是数学不是目测**：同页元素的边缘、间距、图标中心线必须精确对齐——用 grid/flex 的 gap 统一间距，禁止逐个手写 margin 凑位置；固定舞台下所有尺寸都是确定的 px，没有"差不多齐"。
- **一致性守恒律**：圆角使用不超过两档 token；边框使用有语义的 1/2/3/4px 阶梯；动效时长使用同一套语义 token。禁止为单页临时发明新值。
- **CRAP 检查**：对比、重复、对齐、亲密性必须同时成立。标题和正文没有明显对比、相似元素没有重复规则、元素没有数学对齐、相关内容没有聚合，都会直接降低高级感。

## 信息层级与密度自适应

- **极端字阶对比**：每页必须有一个"第一视觉锚点"——它的字号是正文的 3-5 倍（大数字、金句、章节号）。字阶差不够大，页面就没有视线入口；全页字号都在 17-24px 之间徘徊是典型的失败页。
- **单页色彩预算 ≤3**：每页出现的有彩色不超过 3 种（背景色、主文字色不计）。核心信息用唯一强调色"跳"出来，其余全部退到中性色。颜色种类是杂乱感的第一来源。
- **密度随输出模式自适应**（模式定义见 `01-主题内容设计.md`）：
  - 阅读件文字多时：压缩字号一档、行高收到下限、多栏分流，做到"满而不乱"，不砍论证内容。
  - 演示件：单页信息越少字号越大，金句页可以只有一行 52px 以上的字加大留白；不要用阅读件的密度做演示件。
- **视线路径**：观众按 Z 字形/F 字形扫视。锚点放左上或中央，结论和行动放右下（页脚上方），不要把最重要的信息放在左下角。

## 图片与图标

- **图片必须过风格处理，不裸贴**：
  - 纸感风：照片加暖调（`filter: sepia(.12) saturate(.9)` 量级）+ 细边框或纸色衬底，融进米白底。
  - 信息图风：全彩照片会瞬间毁掉黑白红的克制感——照片必须双色调/线稿化处理，或干脆不用照片、改用线框图示。
  - 任何图片都放进容器（边框/衬底/圆角随预设），禁止裸浮在背景上。
- **图标系统**：全套统一线性图标、统一描边宽度（2px）；禁止 emoji 当图标；禁止线性图标和面性图标混用。
- **关键词触发图标**：卡片/章节标题旁的图标按语义匹配（安全→锁/盾、增长→上扬折线、协作→交叠圆、时间→时钟），让观众零点几秒感知区块大意。图标是辅助识别不是装饰——匹配不上语义就不放，禁止硬凑。
- **符号也是系统**：◆ ▶ ✓ ✕ 这类字符标记属于符号系统，同一符号全 deck 含义唯一，且必须在页脚图例里解释。

## 品牌规范（Brandbook）注入

用户提供品牌色、品牌手册或 VI 规范时，按 token 映射注入，不推翻预设骨架：

- **品牌色**：
  - 纸感风：只替换 `--accent`（必要时连带 `--accent-warm`），米白纸底和黑白灰骨架**不动**。
  - 信息图风：红色是这个风格的签名。品牌色接近红/橙时可整体替换朱红并沿用红色纪律；品牌色是蓝绿系时替换藏蓝、保留朱红做唯一强调；不要出现三个以上有彩色。
  - 品牌色本身是渐变（如蓝紫渐变）时，取其主色做单色使用，不引入渐变。
- **品牌字体**：有指定字体时替换 `--font-display`/`--font-body` 的首选项，降级链保留系统字体；无法本地加载时说明降级，不引用不可用的 CDN。
- **Logo**：只放封面信息脚和结尾页，尺寸克制（高度 ≤32px），禁止每页角落都贴 Logo。
- **规范冲突**：品牌规范与预设规则冲突时（如品牌要求大圆角），品牌规范优先，但必须全套统一执行，仍然遵守一致性守恒律。

## 全局细节清单

每套 deck 交付前逐项检查：

- **黑色压边条**（NotebookLM 风）：上下贯通，任何页面不缺席。
- **页脚 meta 行**：左侧写本页版式说明或提示语，右侧页码 `n / N`，13px 弱化色。
- **标注层**：每个图表页至少一层注释——指向箭头、手写式标注、图例、来源行。图例写在页脚（如"◆ 关键决策点 · ▶ 持续推进"）。
- **白色描边分离**：重叠在图表上的独立标记（如甘特菱形）加 `box-shadow: 0 0 0 6px 底色` 白圈，与底下的图形脱开。
- **密度**：图表占画布主体 ≥60%；标题区收紧贴内容。大面积留白只允许出现在封面、原则页、结论页。
- **占位词禁令**：交付前"要素一""说明文字"这类占位词必须全部替换为真实内容。
- **数据真实**：所有数字有来源；来源行写在卡片或页脚。没有来源的数字页改成定性版式。

## 动效气质

- 两个预设共用机制：`data-reveal`（进页自动错峰入场）+ `data-step`（按键分步揭示）。
- 纸感风动效偏"沉"：淡入上移、路径缓绘，420ms ease-out。
- 信息图风动效偏"利落"：条形生长、瀑布块堆叠、数字滚动控制在 600-800ms。
- 任何带 `data-reveal`/`data-step` 的元素**不得依赖自身 `transform` 做定位或旋转**（揭示动画会覆盖 transform）；居中用 inset+margin，旋转放到伪元素上。

## 禁用项

- 科技蓝紫渐变（除非品牌色如此）。
- 默认 AI 紫蓝光晕、三等分卡片、无意义玻璃拟态和装饰性滚动提示。
- "阶段一/阶段二/方向一/看点一"这类泛标签；用具体动作、判断或机制命名。
- 无主题关系的抽象装饰背景。
- 同一页超过一个红色强调信息（信息图风）。
- SVG `<text>`（缩放舞台下渲染错位，文字一律用 HTML）。
- 装饰层用真实 DOM 元素而不排除在通配选择器之外（用伪元素最安全）。
- 阴影堆叠、玻璃拟态、大圆角（信息图风）。
- emoji 当图标；线性/面性图标混用。
- 全彩照片裸贴（必须过风格处理并放进容器）。
- 单页临时发明新圆角、新边框宽、新动效时长（违反一致性守恒律）。
- 中文大标题用负字距。
