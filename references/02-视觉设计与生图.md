# 视觉设计与生图

## 目标

为 HTML 演示和 PPT 输出建立统一视觉系统，并决定哪些页面需要图片生成、哪些页面更适合图表、图标、排版或纯文字。

## 视觉系统

必须定义：

- 气质：专业、科技、财经、教育、发布会、产品、研究、新闻感等。
- 色彩：主色、辅助色、背景色、强调色；避免一套页面全是同一色相。
- 字体：中文标题、中文正文、数字字体的气质说明。
- 版式：封面、章节页、内容页、数据页、案例页、总结页。
- 图形语言：线条、卡片、网格、编号、时间轴、流程箭头。
- 留白规则：保证标题、正文和图像不互相挤压。

视觉系统不能停在“高级、简洁、杂志感”这类形容词，必须同时写清：

- **字阶契约**：大标题/结论、正文、注释/来源三级层级。以 1440×810 舞台为参考，大标题通常 80-96px，小标题/结论 36-44px，正文 24-28px，注释/来源 16-20px；这是比例基线，不是要求每页机械套用固定值。
- **留白契约**：说明页面安全边距、模块间 gap 和有意保留的空白区。留白必须帮助聚焦、分组或制造节奏，不能只是“没有内容”。
- **强调色契约**：默认中性色系 + 一个主强调色。第二强调色只用于明确的对照、警示或状态语义，不能与主强调色同时承担装饰。
- **对齐契约**：标题、正文、图像、图表和来源行归入同一网格；图文页写明垂直对齐、基线或边缘对齐方式。
- **数据契约**：每个数据页只证明一个结论，优先 1-3 个关键数字；数字与说明最多两级主字号，来源行另作注释层。
- **节奏契约**：字体、间距、配色、边框和动效档位前后一致，版式轮廓可以变化，但视觉语法不能每页重置。

### 色彩 token 分层

颜色必须以 token 形式定义并写入 CSS 变量，不要在版式里写死色值：

- 背景两级：页面底色、卡片/面板色。
- 文字三级：主文字、次级文字、弱化文字。
- 强调色默认一个；可预留第二语义色，但只在对照页、警示或状态区出现，不与主强调色竞争。
- 深色页 token：深色底、深色页文字两级，供章节页和结尾页使用。

**禁止默认使用"科技蓝紫渐变"**。这是 AI 生成演示稿最常见的廉价感来源；除非用户品牌色就是蓝紫，或用户明确要求，否则不要用蓝紫渐变背景、蓝紫光晕和蓝紫玻璃卡片。

### 字体配对策略

- 默认策略：中文标题用衬线（`Noto Serif SC`/思源宋体/宋体族降级）制造正式感，正文用无衬线（PingFang SC/思源黑体族），数字使用独立的现代无衬线字体链。
- 必须定义 `--font-number: "Helvetica Neue", Arial, "PingFang SC", sans-serif` 或同等质量的离线字体链，并给纯数字、大年份、序号、百分比和指标使用 `font-variant-numeric: lining-nums tabular-nums`。不要只写 `tabular-nums` 后仍让数字继承 Georgia、宋体或明朝体。
- 标题内出现视觉权重较高的数字时，把数字包进 `.number` / `.metric-number` 独立元素；例如 `<span class="metric-number">50+</span> 人才`。数字与“天、%、亿元”等单位放入同一个 Flex/Grid 容器，用基线或底边对齐，禁止单位孤零零落在下一行。
- 科技、产品类主题可全用无衬线，但标题和正文必须有明显字重与字号层级。
- 字体必须写完整离线降级链；如需 webfont，说明网络依赖和离线替代。

推荐 token：

```css
:root { --font-number: "Helvetica Neue", Arial, "PingFang SC", sans-serif; }
.metric-number {
  font-family: var(--font-number);
  font-variant-numeric: lining-nums tabular-nums;
  font-feature-settings: "lnum" 1, "tnum" 1;
}
```

## 页面图片策略

- 必须生图：封面主视觉、抽象概念、场景氛围、章节视觉锚点。
- 可生图：案例场景、产品愿景、未来场景、用户旅程。
- 不建议生图：严肃数据页、引用页、流程密集页、需要真实截图的页面。
- 禁止伪造：真实品牌截图、新闻现场、真实人物照片、未经提供的 Logo、版权角色。

每页都要能回答“为什么用图”或“为什么不用图”。数据页、流程页、对比页、方法论页优先使用图表、结构图、矩阵、时间线或卡片组，不要用无信息量图片稀释密度。

图文并置时，图片承担场景、证据或结构叙事，文字只负责一句判断和必要解释。先确定图片主体与文字块的共同对齐线，再决定尺寸；禁止图片和文字各自悬浮、视觉重心互不关联。若文字已经完整复述图片内容，应删减文字或换成真正提供证据的图片。

### 图片比例与裁切契约

- **检查型图片**：产品截图、数据图、界面、商品、人物和实物。使用 `data-image-role="inspect"`，必须保留观众需要检查的主体、标题、数值或关键控件。
- **氛围型图片**：只承担情绪和背景，可以使用 `cover`，但主体仍不能被切断。
- 原图比例与容器不匹配时，先调整网格与图片占比；其次在专用 viewport 中做内容保护型裁切；只有必须完整检查时才用 `contain`。不要为了“铺满”切掉内容，也不要为了“完整”把图片缩成无法阅读的附件。
- 内容保护型裁切要先识别安全边缘、主体包围盒和文字避让区。裁掉外围空白或无关导航后，将人工确认过的图片标记 `data-crop-approved`。
- `object-fit: contain` 产生的剩余空间必须进入版式系统，例如截图窗口、状态栏、注释列或背景层，不能留下无意的大块空洞。
- HTML 与 PPT 共用同一最终 viewport；禁止 HTML 用一套裁切、导出 PPT 时再换另一套。

## 生图提示词结构

每个提示词包含：

- 用途：封面、章节页、概念页、场景页、背景图。
- 画面主体：一个明确主体，不堆关键词。
- 视觉风格：摄影、3D、插画、信息图、极简商业、产品渲染等。
- 构图：主体位置、留白方向、文字避让区。
- 比例：默认 16:9；必要时补 1:1、4:3 或宽屏背景。
- 色彩：与视觉系统一致。
- 禁用项：文字、水印、Logo、真实人物、复杂小字、错误手部等。
- 后处理：裁切、压暗、加蒙版、作为背景或图块。

图片内不得承载标题、正文、数据或结论。所有可编辑文字必须保留为 HTML/PPT 文本层。

## HTML 与 PPT 适配

- HTML 可使用轻微视差、淡入、缩放、模糊转清晰。
- PPT 优先使用静态最终图，不依赖复杂网页动画。
- 背景图必须预留安全区域，避免转换后遮挡标题。
- 关键图像需要有替代方案：纯色背景、渐变、图标、图表或占位框。

## 失败路径

- 没有生图工具：输出完整提示词和占位图策略。
- 图片风格不统一：回到视觉系统，重写提示词中的色彩和构图。
- 需要真实素材但用户未提供：标记“需用户提供”，不要编造。
- 图片会影响可读性：降低对比、加蒙版、换成局部图或不用图。
- 生图结果包含文字、水印、Logo 或疑似侵权元素：丢弃或重生成，不要手动遮盖后当作合格素材。

## 验收点

- 每类页面有明确视觉规则。
- 生图页面和非生图页面区分清楚。
- 提示词可直接复制给生图工具。
- 视觉系统能同时服务 HTML 和 PPT。
- 不触碰隐私、版权和真实来源边界。
