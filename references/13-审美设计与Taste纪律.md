# 审美设计与 Taste 纪律

## 何时读取

生成或升级 HTML-PPT 的视觉系统时读取。用户提到"高级""有审美""瑞士风""杂志风""不要模板感""HTML 单文件""主题切换"时必须读取。本文吸收 CRAP 平面设计原则、HTML-PPT 单文件实践，以及 Taste Skill 的反模板化方法。

## 先做 Design Read

生成视觉系统前先写一行设计判断：

`Reading this as: <场景> for <受众>, with <设计语言>, leaning toward <模板/技术策略>.`

中文交付时可写：

`设计判断：这是面向企业运营人员的公开课，适合 NotebookLM 信息图语言，偏高密度结构图 + 克制动效。`

如果设计判断会明显分叉，只问一个问题；能从上下文判断时不要追问。

## 三个旋钮

在 `visual-system.json` 或视觉系统说明里记录三个 1-10 旋钮：

- `DESIGN_VARIANCE`：版式实验度。低值更对称克制，高值更非对称和现代。
- `MOTION_INTENSITY`：动效深度。低值只做 reveal，高值允许封面/hero 的复杂动画。
- `VISUAL_DENSITY`：单页信息密度。低值像画廊，高值像咨询图表。

HTML-PPT 默认值：

| 场景 | DESIGN_VARIANCE | MOTION_INTENSITY | VISUAL_DENSITY |
|---|---:|---:|---:|
| 公开课 / 培训 | 6 | 5 | 6 |
| 商务汇报 / 方法论 | 5 | 4 | 7 |
| 路演 / 发布会 | 7 | 6 | 4 |
| 品牌故事 / 人文演讲 | 6 | 4 | 3 |
| 高管阅读件 | 4 | 3 | 8 |

旋钮不是装饰字段。后续版式、动效、密度都要服从它，落地规则：

- `DESIGN_VARIANCE ≥7`：封面和章节页必须打破左对齐惯性（错位、大字出血、非对称留白）；≤4 则全套走严格对称网格。
- `MOTION_INTENSITY ≥6`：每章至少一页强动效版式（路径绘制/瀑布堆叠/count-up）；≤3 则只保留 `data-reveal` 淡入，砍掉全部 `data-step` 编排。**动效声明 = 动效兑现**：定了高值页面就必须真的动起来，做不到就调低旋钮交付干净的静态版，不交付半残动效。
- `VISUAL_DENSITY` 直接决定每页要点数上限和留白档位：3 = 每页一个主张 + 大留白；7 = 图表 + 标注层 + 状态条全上。演示件（Presenter 模式）在场景默认值上密度 −2、动效 +1。

## CRAP 原则

- **Contrast 对比**：标题、正文、元信息必须有明显字号/字重/颜色差异；每页至少一个第一视觉锚点。
- **Repetition 重复**：字体、颜色、边框、间距、动效曲线全套重复；不要每页换一种小风格。
- **Alignment 对齐**：用 CSS Grid/Flexbox 和固定 gap 对齐；禁止靠手写 margin 目测凑齐。
- **Proximity 亲密性**：相关信息靠近，不相关信息用留白或分割线分开；不要把所有东西装进同样的卡片。

## 六项基础设计契约

这六项是所有模板和风格之上的底层纪律。先让文字、空间和关系完成设计，再考虑装饰和动效。

1. **建立清晰层级**：大标题/结论、正文、注释/来源三级足以覆盖大多数页面。标题必须明显强于正文，正文必须明显强于注释；不要让全页文字挤在同一字号和字重。
2. **留白优先于填满**：留白是信息组织，不是浪费。每块空白都应服务于聚焦、分组、阅读顺序或章节停顿；无法说明作用的大片空白是未完成，不是高级。
3. **色彩保持克制**：默认采用中性色系 + 一个主强调色。强调色只标关键动作、重点数字或唯一决策点；去掉强调色后，页面仍应能靠明暗和字号读出层级。
4. **图文对齐搭配**：图片讲场景、证据或结构，文字提炼判断。图文必须共享网格、边缘或中心线，不得各自漂浮；图片不是填空装饰，文字也不重复图片已经表达的内容。
5. **数据只说重点**：每个数据页只证明一个结论。优先 1-3 个关键数字，数字与说明最多两级主字号；所有数字可追溯，没有来源就改成定性表达或明确标注教学示例。
6. **统一全篇节奏**：字体、间距、配色、边框和动效使用同一套 token 与语义档位。页面轮廓要有变化，但视觉语法不能每页换一套。

生成前把六项写入视觉系统；逐页规划时至少回答层级、留白、强调色、对齐和本页唯一结论。它们是审计条件，不是成品里的说明文案。

## 两个高优先级审美方向

### 瑞士国际主义风

适合商务、产品、数据、方法论。特征：

- 严格网格，建议 12 或 16 列。
- 直角、无阴影、少圆角或 0 圆角。
- 发丝分割线、强字号对比、极简色彩。
- 一个高饱和锚点色，其他退到黑白灰。
- 图片和图表严格对齐网格，标题不要压在图片里。

触发词：专业感、极简、商务、数据、咨询、方法论、瑞士风。

### 电子杂志风

适合观点分享、人文、演讲、品牌故事。特征：

- 大标题、衬线或高质量标题字体、充足留白。
- 横向翻页像杂志跨页，节奏上有 hero、图文页、金句页、案例页。
- 图片是第一公民，比例稳定，和正文区严格对齐。
- 纸白/墨色基调加一个重点色。

触发词：故事感、质感、杂志、观点分享、品牌叙事、公开演讲。

## HTML-PPT 单文件纪律

- 颜色、字号、间距、圆角、动效曲线写进 `:root` CSS 变量。
- 主舞台保持固定 16:9，内部用 CSS Grid 组织。
- 图片不画入标题和页脚；图片只承载视觉，文字用 HTML。
- 键盘左右、空格、触屏点击可用；`B` 键切换静态最终态，便于截图和低性能设备。
- 主题切换只改 token，不重写每页 CSS。
- 静态模式和 `prefers-reduced-motion` 下，所有 reveal 显示最终态。

## Anti-slop 禁令

以下是 HTML-PPT 的常见 AI 味：

- 默认科技蓝紫渐变、紫色光晕、暗色网格背景。
- 三张等宽卡片连续出现，且没有结构关系。
- 每页都有圆角卡片、阴影、玻璃拟态。
- 装饰性标签、无意义状态条、虚构版本号、装饰性地点/时间。
- 英文或中文破折号当万能连接符。
- "阶段一/阶段二/阶段三" 这类泛标签；直接用动作或判断当标题。
- 图片上漂浮标签、伪摄影署名、无来源图片说明。
- 滚动提示、装饰性小圆点、无语义 emoji 图标。
- 过多颜色、过多字体、过多圆角尺度。
- 序号式眉标："00 / INDEX""001 · 能力盘点"式的编号前缀。眉标直接说主题，不用编号装点。
- kicker 不是每页标配：kicker/眉标用于封面、章节页和需要归类的关键页；普通内容页标题本身够用，全 deck 每页同款 kicker 是模板感的主要来源。
- 中点 `·` 配额：元信息行每行最多 1 个中点分隔符，不把 `·` 当万能连接符串一整行。
- 案例页虚构人物用"张三/李四/John Doe"和默认头像；用符合语境的真实感名字，或干脆匿名化（"某消费品牌运营负责人"）。
- 无来源的假精确数字（"92%""4.1 倍"）当真实数据展示；要么给出处、要么标注示例、要么删。

## 生成前检查

生成 HTML 前必须确认：

- 已写 Design Read。
- 已从 `assets/style-library/index.html` 查看真实预览，并记录主模板 ID、辅模板和至少 3 个构图签名。
- 已设置三个旋钮。
- 已说明图片策略：不用图、生成图、用户素材、线框图示或图表。
- 已说明节奏：hero/章节/图表/案例/金句如何交替。
- 已写六项基础设计契约：字阶、留白、强调色、图文对齐、数据聚焦和全篇节奏，不只写风格形容词。

## 生成后检查

交付前至少检查：

- 去掉颜色只看黑白层级，仍然能读出主次。
- 连续 3 页不使用同一种普通卡片结构。
- 每页只有一个核心视觉锚点。
- 每页能在 3 秒内读出标题/结论、正文、注释三级层级；没有整页同权重文字。
- 每个大留白区都有聚焦、分组或停顿作用；没有像漏放内容的空洞。
- 所有关键图片和图表对齐内容网格。
- 默认只有一个主强调色；第二语义色只在明确对照、警示或状态场景出现。
- 每个数据页只服务一个结论，数字与说明没有超过两级主字号。
- CTA、按钮或下载入口文本不换行，且对比度清楚。
- 没有占位词、装饰性文案或无来源数据。
